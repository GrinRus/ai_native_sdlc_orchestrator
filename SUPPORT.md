# Support

AOR is an alpha project with GitHub source and npm CLI alpha package channels.
Support is best-effort and happens through GitHub repository workflows.

## Supported Channels

- Use GitHub issues for reproducible bugs, documentation gaps, and feature requests.
- Use pull requests for focused fixes that follow `CONTRIBUTING.md`.
- Use `SECURITY.md` and GitHub Private Vulnerability Reporting for vulnerabilities or possible secret exposure.
- Use npm package reports only for the published CLI package `@grinrus/aor`.

## Not Supported Yet

- Hosted SaaS operations.
- Public SDK/API support for internal workspace packages.
- Enterprise identity-provider setup.
- Managed multi-tenant operations.
- Production incident response SLAs.

## Response Expectations

There is no guaranteed response time. Maintainers prioritize security reports, public-repo safety issues, contract/source-of-truth drift, and focused pull requests with passing local evidence.
