# User story coverage matrix

This matrix is the machine-checkable story registry for the current 116-story AOR working set. It records current implementation evidence, distinguishes baseline evidence from production proof, and calls out blocked outcomes whose prerequisites remain outside the current self-hosted release scope.

Coverage status values:
- `baseline-covered` - current docs, contracts, examples, and code expose a bounded executable baseline path; no production proof is implied.
- `proof-covered` - executable production-strength evidence exists for the outcome, including real proof artifacts where the story requires them.
- `partial` - current implementation covers part of the outcome, but a backlog slice owns the missing behavior, hardening, or proof evidence.
- `blocked` - the outcome cannot be closed honestly until a named prerequisite or real proof slice lands.

Current W65-S07 status counts: `baseline-covered=99`, `proof-covered=15`, `partial=0`, `blocked=2`. The 114-story W34-S06 result remains a historical pre-audit snapshot. W57-W59 provide bounded self-hosted release clearance through the complete remediation ledger, deterministic wave closures, executable browser and runtime gates, and independent S1 review. W61 adds installed topology onboarding and execution-setup proof; W62 adds deterministic monorepo and bounded-multirepo execution, recovery, integration, delivery, and browser lineage. W63-S08 closes the canonical safe no-write browser lifecycle and W65-S07 re-proves it from the installed single-renderer default; OpenCode certification outcomes remain blocked.
