# Incident report

## Purpose
Durable record of a failed release, production issue, or significant workflow failure that should feed learning memory.

## Required fields
- `incident_id`
- `project_id`
- `severity`
- `summary`
- `linked_run_refs`
- `linked_asset_refs`
- `status`

## Notes
An incident report should be able to backfill a dataset case and trigger recertification.
