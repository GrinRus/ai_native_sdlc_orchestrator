# AGENTS.md

`docs/architecture` is the source of truth for AOR runtime behavior, system boundaries, and operator flow.

## Owns
- target architecture and package map
- execution flow and operating model
- routing, eval, harness, and live-view design
- CLI and control-plane interaction model

## Rules
- Write module-facing docs, examples, and comments in English.
- Keep flows explicit, bounded, and reproducible.
- Prefer canonical lifecycle descriptions over scattered notes.
- When a flow changes, update the matching contracts and examples too.
