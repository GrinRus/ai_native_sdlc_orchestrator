---
name: repo-navigation
description: Use when you need to decide where a change belongs in the AOR repository or which docs, examples, and modules must change together.
---

1. Classify the requested change: product, architecture, contract, example, ops, or implementation.
2. Read the nearest `AGENTS.md` plus the most relevant architecture and contract docs.
3. Identify the minimum matching set of files that must stay in sync.
4. Prefer updating the source-of-truth docs before implementation.
5. Finish by checking whether examples and runbooks now need updates.
