---
name: story-traceability
description: Use when you need to verify that a change maps to supported user stories or when you need to check that a workflow still closes the intended role-based outcome.
---

1. Find the relevant role cluster in `docs/product/00-supported-user-stories.md`.
2. Restate the target user outcome in one sentence.
3. Check which packet, route, policy, contract, and runbook close that outcome.
4. Identify gaps where a story exists but no executable flow exists yet.
5. Update docs or backlog items to close the gap explicitly.
